## Security Gaps

| #   | Severity   | File                                                                                                  | Issue                                                                                                                                                                                                                                                                                  |
| --- | ---------- | ----------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| S1  | **HIGH**   | `useOnboardingStore.ts`, `useAppStore.ts`                                                             | Zero persistence — state resets to defaults on every cold start. Onboarding replays every launch. A returning user can overwrite their hero name/class with no conflict resolution. When persistence is added, plain AsyncStorage is unencrypted and tamper-visible on rooted devices. |
| S2  | **HIGH**   | `src/services/usageStats/UsageStatsService.ts:60`                                                     | `hasUsagePermission()` hardcoded to `return true`. When the native bridge is wired up, `PACKAGE_USAGE_STATS` will be exercised without user grant. Should fail-safe to `return false`.                                                                                                 |
| S3  | **HIGH**   | `src/services/usageStats/UsageStatsService.ts:98-110`                                                 | `__DEV__` 20% random trigger fires in EAS `development` profile builds (internal distribution). Will randomly show Mirror screen to real testers and apply fake stat penalties. Guard with `APP_VARIANT === 'development'` via `expo-constants` instead of raw `__DEV__`.              |
| S4  | **HIGH**   | `src/store/useSessionStore.ts:22`, `src/hooks/useCountdownTimer.ts:38`                                | Timer is 100% client-controlled. `startedAt` is a client-set ISO string in mutable in-memory state. On rooted device: set `startedAt` to 90 min ago → instant full XP. No server-side validation path exists. Architecture must preserve a trusted-clock path for when backend lands.  |
| S5  | **MEDIUM** | `src/services/mock/hero.mock.ts:25-28`                                                                | `mockUpdateHero` doesn't mutate `MOCK_HERO` — uses spread on a `const`. `invalidateQueries` re-fetches original data, reverting all XP/stat changes. Stat decay writes and XP penalties silently disappear. Makes correctness testing impossible pre-API.                              |
| S6  | **MEDIUM** | `src/screens/onboarding/HeroNamingScreen.tsx:171`, `src/services/journal/generateJournalEntry.ts:211` | Hero name has no sanitization beyond `.trim()`. RTL override characters, zero-width joiners, and control characters pass through. When the LLM journal upgrade lands, name is string-interpolated directly into prompts — prompt injection vector.                                     |
| S7  | **MEDIUM** | `src/navigation/RootNavigator.tsx:10`                                                                 | Onboarding guard reads non-persisted store. `completed` is always `false` on cold start — onboarding replays every launch. A user can submit a new class/name and overwrite their hero with no guard.                                                                                  |
| S8  | **MEDIUM** | `src/hooks/useStatDecay.ts:64`                                                                        | `if (permissionGranted \|\| __DEV__)` bypasses the permission gate in all dev builds. The inner `Platform.OS !== 'android'` check saves iOS for now, but the bypass is fragile. Remove `\|\| __DEV__` — the dev fixture in `checkBlacklistUsage` is sufficient.                        |
| S9  | **LOW**    | `src/screens/focus/FocusScreen.tsx:127`                                                               | `handleSurrender`'s `mutateAsync` call is not inside try/catch. Unhandled promise rejection on mutation failure.                                                                                                                                                                       |
| S10 | **LOW**    | `src/hooks/useCountdownTimer.ts:41`                                                                   | Uses `Date.now()` wall-clock delta. Device clock moved backward resets elapsed time; moved forward completes session instantly. Use `performance.now()` for monotonic timing.                                                                                                          |
| S11 | **LOW**    | (no file)                                                                                             | No `ErrorBoundary` anywhere in the component tree.
